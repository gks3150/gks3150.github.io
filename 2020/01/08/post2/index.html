HTML/CSS 공부하며 적은 메모

  1. HTML class 명명에는 숫자로 시작할 수 없다.
    <p class="1_sidebar"></p> 와 같은 방식의 class 명명은 제한된다는 것

  2. <div>는 블록 요소(block level elements)이며 컨텐츠 크기를 포함한 box 형태이며 한 줄로 배치가 된다.
    블록 요소는 인라인 요소와 다르게 padding border margin 값을 줄 수 있다.

  3. 기본 스타일 속성 초기화의 중요성!! 안하면 나중에 곤란할 일이 많이 생길 수 있다.

  4. CSS 선택자 간 우선 순위를 알아두어야 함.

  5. image는 블록 요소가 아니다. content elements이다.
    부모요소가 content align하면 부모에게 지정된 자식 요소들 전부 align된다. (상속)

  6. a 태그의 경우 display 속성으로 inline 속성인데 block 속성으로 바꿀 수 있음.
    인라인 요소를 가지는 태그는 어떤 것이 있는가?
    <a><span><strong><em>
    width, height 지정 불가, 무조건 컨텐츠 크기 만큼으로 자동조절.
    padding, margin은 좌우만 적용 가능.
    padding의 상하단은 시각적인 부분만 적용된다.

    블록 요소를 가지는 태그는 어떤 것이 있는가?
    <div><h1><p><ul><li>
    width,height,padding,margin 모두 적용 가능.

    inline-block 태그에는??
    <img>
    inline과 block의 장점들을 고루 가지고 있음.
    부모에게 text-align 적용 가능, 언제나 컨텐츠 크기 만큼 조절됨.
    margin, padding은 block과 같이 동작함.

  7. 가운데 정렬 태그 스타일에 margin: 0 auto; 값을 준다.

  8. NAME:hover -> 가상클래스라고 한다.

  9. .NAME::before,after -> 가상요소 기본적으로 인라인 요소라는 것 기억할 것.

  10. font-size: 16px이라 가정하면 block의 전체 높이는 16px이 아님. half reading 영역이 존재한다.
    line-height는 block 까지의 거리 조절
    line-height:1이면 font-size와 block의 크기 동일하게 수정한다는 것을 의미. 섬세한 작업이 가능하다.

  11. float 속성은 뒤따라오는 컨텐츠와 상관없이 공간을 효율적으로 사용하도록 float 속성을 가진 컨텐츠와 섞이게 만들어 준다.

  12. clear:both; 뒤따라오는 컨텐츠를 float에 영향을 받지 않도록 뒤따라오는 컨텐츠에 추가해주는 속성.

  13. display: flow-root 대신 overflow: hidden을 사용할 수도 있다. IE 호환을 위해서…

  14. sidebar가 오른쪽에 있을 경우 시각적으로 봤을 때, 마크업 순서가 contents 다음에 sidebar가 와야 한다고 착각하기 쉬운데 그렇지 않다.

  15. width: auto; 부모 태그의 너비 값을 상속받아 사용한다는 의미.

  16. margin-left: auto 활용하지 않은 공간을 margin-left로 모두 취하겠다는 auto의 의미.

  17. margin: 0 auto의 경우 left,right 값을 활용하지 않은 공간을 기준으로 모두 취하겠다는 의미니까
    가로를 기준으로 가운데 정렬과 마찬 가지의 의미임.

  18. box-sizing: border-box; 최신 css 속성. width height로 잡아 놓은 box모델이 padding값 추가해도 변하지 않게 해줌.

  19. * -> 와일드 카드, universial selector 모든 태그를 의미함. 권장하진 않음.

  20. negative margins는 box의 공간을 차지하지 않고 시각적으로 빠져나간 부분만 보여주는 역할을 한다.

  21. 반응형 웹이란? 사용자 브라우저가 어떤 것이든 상관없이 최적의 레이아웃을 제시하는게 가능하도록 하는 것. 사용자 창 크기가 달라졌을 때와 관계없이, 모든 컨텐츠가 보여질 수 있도록 적절히 레이아웃을 변경하도록 하는 것.

  22. max-width와 % 사용하는게 현명하다. 고정 px을 사용할 경우 화면 크기에 따라 레이아웃이 달라질 수 있음.

  23. <meta name="viewport" content="width=device-width, initial-scale=1.0"> 뷰 표준 태크, 스마트폰에서도 동일한 화면을 보게 하도록 하기 위해서 작성해주는 태그

  24. CSS 방법론에 BEM, OOCSS, SMACSS가 있다.
